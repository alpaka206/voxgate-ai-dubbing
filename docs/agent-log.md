## 2026-03-14

- 타임스탬프 기반 더빙으로 바꾸면서 세그먼트마다 Gemini 번역 요청이 발생해 free tier quota 초과가 쉽게 재현되었고, 요청 수를 줄이는 배치 번역과 `429` backoff 처리가 추가로 필요해졌습니다.

- 배포된 Vercel 환경에서 11MB 파일 업로드가 `413 Content Too Large`로 실패했고, 원인은 앱 내부 50MB 제한보다 앞단의 Vercel Function body limit(4.5MB) 제약이 먼저 적용되기 때문이었습니다.
- 단순히 코드 상수만 늘려서는 해결되지 않고, 업로드 본문과 결과 파일을 함수 본문으로 직접 주고받지 않는 구조로 바꿔야 실제 서비스에서 50MB 범위를 처리할 수 있어 변경이 필요했습니다.
- 기존 더빙은 전체 전사를 한 번에 번역하고 음성을 한 번만 합성해 원본의 중간 pause와 발화 길이를 반영하지 못했고, 비디오 합성도 더 짧은 오디오 길이에 맞춰 결과 길이가 줄어드는 문제가 있어 타이밍 보존 방식으로의 변경 이유가 추가되었습니다.
- 랜딩 페이지 피드백에서 Google 로그인 버튼의 포인터 상태와 시각 규칙, 페이지 전환 시 전역 로딩 부재가 지적되어 로그인 버튼 스타일과 App Router 전역 로딩 UI를 보강할 필요가 생겼습니다.

## 2026-03-17

- 대용량 업로드 대응 요구사항에 맞춰 브라우저가 업로드 전에 원본 길이를 확인하고, 60초를 넘는 파일은 첫 1분만 잘라 업로드하도록 스튜디오 흐름을 변경했습니다.
- 60초 이하인데도 `50MB`를 넘는 오디오/비디오 파일은 브라우저에서 모바일 업로드용 MP4/M4A로 다시 압축하도록 클라이언트 `ffmpeg.wasm` 준비 단계를 추가했습니다.
- 입력 원본은 `WORKERFS`로 브라우저 워커에 마운트하고 출력된 업로드용 클립만 메모리로 읽도록 구성해 일반적인 모바일 폰에서도 원본 전체 복사를 피하도록 조정했습니다.
- 스튜디오, 대시보드, 마이페이지 문구를 `브라우저 1분 clip` 기준으로 다시 맞추고, 서버 처리 한도(`3분`)와 업로드용 클립 정책을 분리해 안내했습니다.
- `README.md`, `.env.example`, `docs/agent-log.md`를 새 업로드 정책 기준으로 갱신했고, `npm run lint`, `pnpm -s tsc --noEmit`, `npm run build`로 회귀를 확인했습니다.
